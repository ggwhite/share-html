安裝指南

從零開始，三步驟完成安裝與初始化。

terminal 
# 1. 下載專案
git clone https://github.com/example/project.git
cd project

# 2. 安裝依賴
npm install

# 3. 啟動開發伺服器
npm run dev
INFO

Node.js 18+ 為必要條件。可透過 node --version 確認目前版本是否符合需求。

設定說明

常見問題與進階設定選項。

如何修改預設連接埠？ ›
在專案根目錄建立 .env 檔案，加入 PORT=3001。若使用 Docker，請同時更新 docker-compose.yml 中的 ports 設定，確保 host 與 container 兩端的埠號一致。
如何在本機啟用 HTTPS？ ›
本機開發建議使用 mkcert 建立受信任的本機憑證，並設定 SSL_CERT 與 SSL_KEY 環境變數指向對應憑證路徑。正式環境建議在反向代理層（Nginx / Caddy）終止 TLS，應用層維持 HTTP。
WARN

環境變數中不應存放明文密鑰。請使用密鑰管理工具（如 Vault、AWS Secrets Manager）統一管理敏感設定。

API 參考

常用端點說明與範例回應。

GET /api/v1/users/:id

取得指定使用者的詳細資訊。需要 Bearer token 驗證。

Response 200 
{
  "id": "usr_01H8...",
  "name": "白小明",
  "email": "white@example.com",
  "role": "admin",
  "created_at": "2026-01-15T08:30:00Z"
}
POST /api/v1/users

建立新使用者。需要 admin 權限。

Request Body 
{
  "name": "新使用者",
  "email": "new@example.com",
  "role": "viewer"
}
DELETE /api/v1/users/:id

刪除指定使用者。此操作不可逆，需要 admin 權限。

DANGER

刪除操作會同時移除該使用者的所有關聯資料，包含操作紀錄與權限設定。
