引言

為何 HTML 在 LLM 時代取代 Markdown 作為輸出格式。

“

Markdown is a report. HTML is an interface.
— Markdown 是一份報告，HTML 卻是一個讓人繼續工作的介面。

— Thariq Shihipar · Anthropic Claude Code 團隊

當 LLM 一次能產出數萬字結構化內容，過去選擇 Markdown 的理由（短、輕、純文字）已不再是瓶頸。把輸出做成「能用」而非「能讀」，是這個時代的合理升級——HTML 支援互動、狀態追蹤、即時搜尋與篩選，這是純文字格式無法企及的。

2

能力對照

兩種格式的根本差異——「動 vs 不動」。

MD

Markdown 的限制

  • ·純粹線性閱讀，無法折疊或篩選
  • ·無法嵌入互動 widget、SVG 圖表
  • ·長報告需要人工捲動、複製、剪貼
  • ·沒有狀態（待辦／已完成／優先級）
  • ·無法搜尋、無法即時更新計算
{ }

HTML 帶來的能力

  • ·SVG 圖表、互動 widget、頁內導覽
  • ·可折疊、可篩選、可排序
  • ·狀態徽章、優先級顏色、進度條
  • ·同份輸出既是檢視也是工作面板
  • ·可加入 JS 計算、即時試算、模擬

進階說明

把次要 / 較長的內容折疊起來，需要才展開。

為什麼選擇參數化查詢而非 escape ›
escape 需要針對每種 driver、每種 quote 格式處理，容易漏；參數化由 driver 統一處理，覆蓋所有 edge case。即使 escape 寫對也建議改成參數化，降低後續維護風險。
Bearer token exp 檢查的 grace period 怎麼設 ›
建議 5 秒寬限以容忍 clock skew；過長會讓被竊取的 token 多一段風險視窗，過短則容易誤判合法請求。

程式碼塊

highlight.js 自動上色 + 右上複製按鈕。

auth.ts 
async function authenticate(token: string) {
  const decoded = jwt.verify(token, SECRET);
  if (decoded.exp < Date.now() / 1000) {
    throw new UnauthorizedError('token expired');
  }
  return decoded;
}
